# Design: Cmd/Ctrl+K Palette v1.1 ŌĆö Albums, Spaces, and Truncation Indicator

**Date:** 2026-04-16
**Status:** Draft ŌĆö two review rounds applied (code-reviewer)
**Builds on:** [`2026-04-12-cmdk-search-design.md`](./2026-04-12-cmdk-search-design.md) (v1, shipped as PR #349) and [`2026-04-13-cmdk-navigation-design.md`](./2026-04-13-cmdk-navigation-design.md) (v1 navigation sub-sections)
**Scope:** Web. Two v1.1 items from the v1 deferred backlog.

---

## Goal

Extend the v1 palette with:

1. **Albums** and **Spaces** as two new sections in the streaming fan-out, with full preview-pane support on desktop.
2. A **"├Ś N more" truncation indicator** on sections lacking a see-all target ŌĆö the two new collection sections, plus the four navigation sub-sections (System Settings / Admin / User Pages / Actions).

## Non-goals (v1.1)

- **Server-side album/space search endpoints.** Client-side filter over a cached list is sufficient at expected scale. Expected payload: `AlbumResponseDto` via `mapAlbumWithoutAssets` (no full asset arrays); `SharedSpaceResponseDto` already includes `recentAssetIds[]`, `recentAssetThumbhashes[]`, member summary, counts.
- **Description / notes matching.** Name only (consistent with Tags and People).
- **Clickable chip.** v1.2 prefix scoping (`/album hawaii`) is the proper answer to "show me only albums"; the chip stays informational.
- **Cross-tab cache invalidation** (`cmdk.albums.version` / `cmdk.spaces.version`). No writers ready; ship palette-close/reopen as the refresh boundary (same as v1 tags in practice).
- **Open-in-new-tab (Ctrl/Cmd+Enter).** Inherited v1 limitation ŌĆö `Command.Item.onSelect` doesn't expose modifier state. Not a v1.1 regression.
- **SUGGESTED empty-state, static map tile, Add-to-album action.** All remain deferred.

## User-facing behavior

### Section order (accepted tradeoff)

```
PHOTOS ŌåÆ ALBUMS ŌåÆ SPACES ŌåÆ PEOPLE ŌåÆ PLACES ŌåÆ TAGS ŌåÆ nav sub-sections
```

Collections sit adjacent to Photos because they answer the same question ("show me stuff about X") more than attribute-matches (People / Places / Tags) do. Per-section cap is `topN=5` with no global cap. Worst case ŌĆö every collection slot filled ŌĆö People begins at row ~16 on a ~12-row visible palette and requires scrolling. Accepted, because:

- Cold queries rarely hit 5 matches in all three collections sections.
- v1.2 prefix scoping is the proper escape hatch for users who already know the entity type they want.

### Matching, ranking, activation

**Match:** case-insensitive substring on `album.albumName` / `space.name` only. Description and other metadata are ignored.

**Normalization:** `query.trim().toLowerCase()` compared against `name.toLowerCase()` using `String.prototype.includes()`. Leading/trailing whitespace trimmed; internal whitespace preserved (so `"hawaii  2024"` literally matches `"Hawaii  2024"`, not `"Hawaii 2024"`). Regex special characters are safe ŌĆö `.includes` treats them as literals.

**Minimum query length:** 2 characters (same as People / Places / Tags). A single character `"a"` would dominate the result list via startsWith matching and produce no signal; gate at Ōēź2.

**Rank:** `startsWith(query) ? 2 : 1`, alphabetic tiebreak. Slice to `topN=5`. Pre-slice matched count flows into `ProviderStatus.ok.total` so the existing truncation trigger (`status.total > status.items.length` at `global-search-section.svelte:44`) drives the chip and heading suffix.

**Enter activation:**

- Album ŌåÆ `Route.album({ id })`, write to `cmdk.recent` as:
  ```ts
  {
    kind: 'album';
    id: `album:${albumId}`;
    albumId;
    label;
    thumbnailAssetId;
    lastUsed;
  }
  ```
- Space ŌåÆ `Route.viewSpace({ id })`, write to `cmdk.recent` as:
  ```ts
  {
    kind: 'space';
    id: `space:${spaceId}`;
    spaceId;
    label;
    colorHex;
    lastUsed;
  }
  ```

The composite `id` (`album:ŌĆ”` / `space:ŌĆ”`) follows the verified v1 convention from `cmdk-recent.spec.ts:40` (`{ kind: 'photo', id: 'photo:abc', assetId: 'abc', ŌĆ” }`). `id` is the stable dedupe key used by `removeById`; the raw underlying id lives as a named field for navigation.

**Stale-entry guard ŌĆö applies to ALL activations, not just RECENT.** Every Enter-on-row path (fresh result OR RECENT) wraps navigation in a resolution check. Rationale: mid-session cache can go stale (another tab deletes a shared album while our cache is 5 minutes old), and a silent navigate-to-404 is the same bug in both paths. One handler; uniform UX.

Implementation shape:

```ts
async function activateAlbum(id: string) {
  const key = `album:${id}`;
  if (activationInFlight.has(key)) return; // double-Enter guard
  activationInFlight.add(key);

  const pendingTimer = setTimeout(() => {
    // 200ms pending affordance
    pendingActivation = key;
  }, 200);

  try {
    // closeSignal aborts on palette close(); does NOT abort on batch rotation
    // (new keystroke) ŌĆö activation must survive subsequent typing.
    await sdk.getAlbumInfo({ id }, { signal: closeSignal });
    if (closeSignal.aborted) return; // escape-during-resolve guard
    goto(Route.album({ id }));
  } catch (err) {
    if (err.name === 'AbortError') return; // palette close ŌĆö silent
    if (err.status === 404 || err.status === 403) {
      cmdkRecent.removeById(key); // no-op if not in RECENT
      toastManager.warning($t('cmdk_toast_album_unavailable'));
      return;
    }
    // 401 and all other errors propagate to the global SDK auth interceptor,
    // which handles redirect-to-login. Do not swallow silently.
    throw err;
  } finally {
    clearTimeout(pendingTimer); // prevent late timer mutation
    activationInFlight.delete(key);
    if (pendingActivation === key) pendingActivation = null;
  }
}
```

Space path analogous via `sdk.getSpace({ id })`. `cmdkRecent.removeById` is a no-op when the entry isn't present, so calling it unconditionally on fresh-result 404s is safe.

**Concurrency and cancellation (this path):**

- **Two abort lifecycles, not one.** `batch.signal` aborts on every keystroke debounce fire (v1 pattern ŌĆö fan-out is per-keystroke). `closeSignal` aborts only on palette `close()`. Activation binds to `closeSignal` because Enter must survive subsequent keystrokes (the user's input focus stays on the combobox, and v1 does not block typing during activation). Fan-out providers continue to bind to `batch.signal`.
- **Double-Enter guard.** `activationInFlight: Set<string>` keyed by composite id. A second Enter on the same row while the first is in flight is ignored; two different rows in quick succession each get their own resolve but only the winner's `goto()` lands (the later `goto()` supersedes).
- **Escape-during-resolve guard.** `closeSignal` aborts on `close()`. Post-await, we re-check `closeSignal.aborted` before `goto()` so a late-landing promise doesn't navigate against a closed palette.
- **401 not swallowed.** The local catch handles 404/403 only. 401 and other status codes re-throw so the global auth interceptor can redirect.
- **Pending-timer cleanup.** The 200 ms `setTimeout` is cleared in `finally` ŌĆö otherwise a late-firing timer mutates state after the activation is done.

**Pending-row affordance.** After 200 ms without resolution, the activated row renders in a disabled-pending style (slight opacity + spinner glyph on the right). Prevents the "Enter did nothing" perception on slow networks. Cleared on resolve (either success ŌåÆ navigate, or catch ŌåÆ toast). Threshold matches v1's progress-stripe grace window.

**Reopen during activation.** If the user closes and reopens the palette while an activation is in flight, `close()` aborts `closeSignal` ŌĆö the in-flight resolve rejects with `AbortError` and falls through the silent return. The `finally` block still runs on the rejected promise, clearing `activationInFlight[key]` and any pending timer. On reopen, `open()` creates a fresh `closeController`; a re-Enter on the same row is allowed (the `activationInFlight` guard sees an empty set) and runs to completion in the new session.

**`goto()` unhandled rejection (inherited v1 behavior).** `goto(Route.album({ id }))` is called without `await` or `.catch()` ŌĆö if SvelteKit navigation fails, the rejection propagates unhandled. This matches v1's activation handlers for photos/people/places/tags; not a v1.1 regression. Worth noting but not worth diverging from v1's pattern.

**Row prop for pending state.** `album-row.svelte` and `space-row.svelte` accept a new prop `isPending: boolean`, derived by the parent from `service.pendingActivation === item.id`. The row applies the pending styling when true. No prop threading through section wrapper ŌĆö pass directly at render time in `global-search.svelte`.

Mirrors the stale-admin-recent render-time pattern from the navigation design (lines 166ŌĆō175), extended to the activation path.

**Toast API:** `toastManager` from `@immich/ui` (verified usage at `web/src/lib/utils/file-uploader.ts:117`). Methods used here: `.warning($t(key))`.

**Stale labels (accepted):** `cmdk.recent` stores a snapshot of `label` at activation time. Renaming an album or space in another tab does not refresh existing RECENT entries ŌĆö old labels persist until the entry is re-activated (which overwrites). Matches how v1 person / place entries already behave.

### Preview pane

Both previews inherit v1's format: ~240px pane width, GoogleSans 14/500 name, GoogleSans 12/410 `text-gray-500 dark:text-gray-400` metadata row, single ghost "Open" button.

**Album preview:**

- Cover thumbnail via `albumThumbnailAssetId` through `createUrl()` (per `feedback_filter_thumbnail_createUrl`).
- `albumThumbnailAssetId === null` (empty album) ŌåÆ placeholder photo icon at pane dimensions.
- Metadata row: `"87 items ┬Ę Mar 2024 ŌĆō Mar 2026"` built from `assetCount`, `startDate`, `endDate`. Date range is omitted when either endpoint is missing.
- Small "Shared" pill badge beneath the name when `shared === true`.

**Space preview:**

- `<SpaceCollage>` (reused from `space-card.svelte:3`) fed by `recentAssetIds` + `recentAssetThumbhashes`.
- Empty `recentAssetIds[]` ŌåÆ gradient-card fallback using the color-to-gradient mapping at `space-card.svelte:24ŌĆō37`.
- Metadata row: `"3 members ┬Ę 420 photos"` from `memberCount` + `assetCount`. ICU plural on `member` handles singular ("1 member ┬Ę ŌĆ”").
- Member avatar strip: up to 4 + `+N` overflow bubble (reuses `visibleMembers` / `overflowCount` pattern from `space-card.svelte:45ŌĆō46`). When the user is the only member, the strip shows just their avatar rather than rendering empty.
- `lastContributor` **intentionally omitted** ŌĆö diverges from `space-card.svelte:52`. Palette preview is a quick-scan surface, not an activity feed. Flagged as an open implementation question; default stays omit.

### "├Ś N more" truncation indicator

Applied to any section where `status === 'ok'` AND `status.total > status.items.length` AND `onSeeAll` is absent. Both cues render together when truncated:

1. **Group heading suffix.** Heading text becomes `ALBUMS (5 of 8)` when truncated, `ALBUMS` when not. i18n key `cmdk_section_count_of`: `{shown, number} of {total, number}`.
2. **Chip row** after the last `Command.Item`: `<div aria-hidden="true">├Ś 3 more</div>`. i18n key `cmdk_section_more_count` uses ICU plural: `{count, plural, one {├Ś # more} other {├Ś # more}}` so pt-BR / Slavic locales can diverge (English collapses to one form).

**Gating:** the chip and heading suffix render only when `status === 'ok'`. During `'loading'` (skeletons visible) or `'error'` (error row), both are suppressed ŌĆö otherwise a skeleton state could briefly show `"5 of 0"` before results land.

**Navigation state-model translation.** `GlobalSearchNavigationSections` uses a synchronous filter-over-static-list; it doesn't expose the async-provider `ProviderStatus` shape. For the nav path the equivalent predicate is `hasResolvedOnce && truncated` (where `hasResolvedOnce` is true after the first keystroke has completed its filter pass ŌĆö no async loading state exists for nav). Document this translation in the component; tests assert chip absence on first-keystroke frame if a jitter window exists.

**Accessibility:**

- Chip is **not** a `Command.Item` ŌåÆ not in Bits UI's item registry ŌåÆ arrow keys skip it automatically ŌåÆ no `role="option"`.
- Count lives in the **visible heading text** itself, not an `aria-label` override. Screen reader announces `"Albums, 5 of 8"` **once** on group entry (VoiceOver's `aria-live` per-item re-announcement behavior was the reason to reject an `aria-label`-only plan).
- `aria-hidden="true"` on the chip prevents double-announcement.

**Applied to:**

- New: Albums, Spaces sections.
- Existing (polarity flip): System Settings, Admin, User Pages, Actions nav sub-sections. **Pre-slice count must be tracked per sub-section**, not as a single flat-list aggregate, because each sub-section slices independently to `topN=5`. Thread pre-slice counts per sub-section through `GlobalSearchNavigationSections`.

## Data flow

### First palette open per session

Three parallel fetches, all bound to **`closeSignal`** (not `batch.signal`) ŌĆö they must survive keystroke-driven batch rotation:

1. `getAllAlbums({})` ŌåÆ owned albums (maps to `albumRepository.getOwned`).
2. `getAllAlbums({ shared: true })` ŌåÆ shared-with-me albums (maps to `albumRepository.getShared`).
3. `getAllSpaces()` ŌåÆ spaces the user is a member of.

**Merge and dedupe albums.** Responses from (1) and (2) are concatenated and deduped by `id`. Dedupe is **load-bearing** ŌĆö an album owned by the user that has also been shared out can appear in both `getOwned` and `getShared`. The merge must survive an implementation-time audit of the repository layer.

**Dedupe keeper preference: prefer the `getShared` record.** When the same album id appears in both responses, the shared-response record wins because it carries the more-informative `shared: true` flag (drives the "Shared" preview badge). Implementation: dedupe by building a `Map` keyed by `id`, seeded first from the owned list then overwritten by the shared list. Unit-test the preference explicitly.

**Abort on close.** Palette `close()` calls `closeController.abort()` AND `batch.abort()`. In-flight oazapfts fetches reject with `AbortError`; the catch path leaves caches unset; next `open()` re-fetches under a fresh `closeController`. Aborted promises must not write into a new cache ŌĆö the service's `open()` path establishes a new cache-ref + closeController-ref + batch-ref atomically, and aborted promises' `.catch` branches are pure no-ops.

### Keystroke path

`runAlbums()` / `runSpaces()`:

1. Await the in-flight cache fetch if it hasn't resolved yet (read from a shared promise).
2. Filter the cached array by the normalized query.
3. Sort by score (`startsWith=2, substring=1`), alphabetic tiebreak.
4. Slice to `topN=5`.
5. Emit `ProviderStatus.ok` with `total = pre-slice matched count` and `items = sliced`.

**Synchronous filter over cached array** ŌĆö no per-keystroke `AbortController` required. Debounce is inherited from v1's batch dispatch.

### Cache lifetime

- Cleared on `close()`.
- Next `open()` re-fetches from scratch.
- No cross-tab invalidation (see Non-goals).

### Error handling

- **`AbortError` does NOT trigger `'error'` state.** Aborted fetches (palette close, signal cancel) are silently dropped. All other failures transition the section to `'error'`.
- Initial fetch 4xx / 5xx ŌåÆ `sections.albums.status = 'error'` (or `.spaces`). Render inline one-row error. Other sections unaffected.
- **401 on fetch** ŌåÆ same `'error'` state locally, AND the global SDK auth interceptor handles redirect-to-login. Do not fall back to stale cache on 401.
- **401 on activation resolve** ŌåÆ re-thrown to the global auth interceptor (see activation code above). Not locally swallowed.
- `albumThumbnailAssetId === null` ŌåÆ placeholder icon in preview.
- `recentAssetIds[]` empty ŌåÆ gradient-card fallback for space preview.
- Space member strip with only current user ŌåÆ render self, not empty.

## Open at implementation time

### Must resolve before coding

- **Thumbnail side-effect.** `album.service.ts:43ŌĆō74` `getAll()` unconditionally calls `albumRepository.updateThumbnails()`. Firing on **both** parallel album calls per palette open is unacceptable. Default approach: new lightweight endpoint `GET /albums/names` returning `{ id, albumName, albumThumbnailAssetId, assetCount, startDate?, endDate?, shared }[]` with no thumbnail update and no full DTO. Palette is the only caller; implementation stays encapsulated. Alternatives (less preferred): `skipThumbnailUpdate` query param (leaks implementation detail); accept the cost (measure first).
- **Dedupe keeper preference verified.** Design specifies the shared-response record wins. Verify at implementation time that the `getShared` response actually carries `shared: true` (the DTO field must be populated by the server path, not only by client inference).
- **Fresh-result stale guard extension.** Design extends the stale-entry guard to all activation paths (not just RECENT). Implementation must wrap every album/space activation in a resolve-before-navigate call. Adds one SDK round-trip per Enter ŌĆö acceptable latency cost for eliminating silent-404 class of bugs.
- **Activation 401 interception.** 401 from the activation resolve must not be swallowed by the local catch. Verify Gallery's SDK client has a global auth interceptor that redirects on 401; if it doesn't, the activation handler must invoke `authManager.logout()` (or equivalent) explicitly before re-throwing. Crash-class bug if left silent.
- **Logout mid-session behavior.** When user logs out in another tab while palette is open, next keystroke's provider call returns 401. Design routes 401 to `'error'` state. Verify the auth guard also redirects the open tab to login, OR explicitly close the palette on logout event. Pick one and document.
