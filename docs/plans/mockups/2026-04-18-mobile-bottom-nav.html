01 — Primary state

Dark surface, amber glow.

The pill floats 26 pt above the home indicator; the search bubble rides the same baseline. Tap a tab and the amber glow slides with a 280 ms spring, icon fades in, label shifts to the accent hue. Inactive tabs keep their label (no icon) — the primary row stays readable at a glance.

Try it

Tap any label in the frame to switch active state. The pill slides, the icon fades in, and haptic feedback fires in the Flutter build.

Why amber, not blue

Google Photos leans on clinical tech-blue. Gallery is forked for a warmer, photography-first audience, and the nav is its most-seen surface. Sodium-lamp amber ties the nav to the brand's darkroom lineage and gives the active state a glow that reads as warm, not flat.

In production the color resolves from theme.colorScheme.primary — whatever preset the user picked in settings. Amber is the demo swatch.

Three visible stops

  • Photos — main timeline (unchanged)
  • Albums — existing Immich albums page re-linked from the bottom nav (default target DriftAlbumsRoute)
  • Library — existing Gallery library page (DriftLibraryRoute); replaces today's Spaces tab. Spaces surfacing inside Library is a follow-up design, out of scope here.
  • Search — not a tab; a peer blob. Tap routes to Photos (if elsewhere) and opens the FilterSheet at Browse with the text-pill focused.

05 — Edge states

What happens when reality intrudes.

The nav is not always on-screen. It hides for multi-select, hides when the keyboard is up, and folds into a rail in landscape — inheriting today's tab_shell behaviour, not inventing new rules.

Keyboard open

Slide out 12 pt + fade. The nav is decorative while the user types; re-emerges on focus loss.

Multi-select active

Listens to MultiSelectToggleEvent and hides — identical behaviour to upstream _BottomNavigationBar. The existing contextual action bar takes over.

(nav hidden · selection bar owns the bottom)

Landscape

Fall back to a vertical NavigationRail on the leading edge — same as today. The pill aesthetic is portrait-first; rail keeps the fork's existing landscape contract intact.

Readonly mode

Disabled tabs dim to 30% and refuse taps, matching the existing isReadonlyModeEnabled contract. Only Photos stays live when readonly.

07 — Decisions locked

What the nav actually does.

Two questions surfaced during brainstorm; both resolved with the smallest-change-that-works instinct the fork has been running on.

Albums → existing Immich page

The Immich albums page was de-linked from the bottom nav in PR #116 but never deleted. The new nav re-links it — no new landing page, no rename.

  • Target route: DriftAlbumsRoute (Drift-based, aligns with the rest of the tab shell).
  • Legacy AlbumsRoute (Isar) remains as a fallback option if Drift parity isn't reached at ship time — call out at PR-draft.
  • Spaces (ex-tab in the current shell) steps out of the bottom nav. Surfacing Spaces inside DriftLibraryPage is out-of-scope for this work and tracked as a follow-up.

Search blob → Photos + FilterSheet

Tap behaviour is deterministic regardless of the current tab:

  • If not on Photos: navigate to MainTimelineRoute.
  • Open the FilterSheet at the Browse snap (shipped in PR 1.3).
  • Focus the text-search pill inside the sheet; request keyboard. The sheet's existing debounce handles input.
  • DriftSearchPage is no longer a tab; it remains a reachable route for anything in-app that deep-links to it.
